Beedeedee

A multithreaded library for binary decision diagrams

Download sources .zip Download sources .tar.gz Download JAR View on GitHub

BeeDeeDee is a Java BDD library that allows concurrent manipulation of Binary Decision Diagrams, by exploiting multithreading.

Binary Decision Diagrams (BDDs) are efficient representations of Boolean formulas. Once a variable ordering is defined, these representations are canonical. That is, two equivalent formulas are represented by the same diagram, and equivalence tests run in constant time. Moreover, efficient algorithms exist for common Boolean operations. By using a BDD library to manipulate Boolean functions, your application will be able to manage big logical formulas easily.

Features

BeeDeeDee allows programmers to use a single factory object, shared between different concurrent threads in an application. The classic problem for testing a BDD library is the N-queens problem: finding a placement of N queens on a NxN chessboard, so that no two queens attack each other. A code for running a number of instances of the N-queens concurrently is listed below: 

// create the shared factory
Factory factory = Factory.mkResizingAndGarbageCollected(utSize, cacheSize);

Set<QueensThread> queens = new HashSet<QueensThread>();

// start threads - 'ens' contains size of n-queens problems
for (Integer n : ens) {
  QueensThread queen = new QueensThread(n);
  queens.add(queen);
  queen.start();
}

// wait for threads to terminate
for (QueensThread queen: queens)
  try {
    queen.join();
  }
  catch (InterruptedException e) {}

// shutdown the factory
factory.done();

/**
* Worker thread. Does actual computation using the shared factory.
*/
private static class QueensThread extends Thread {
  ...
  public void run() {
    // compute n-queens assignment...
  }
}

The interface of a Factory provides methods to create basic BDDs:

  • makeOne() creates the constant One
  • makeZero() creates the constant Zero
  • makeVar(i) creates the variable indexed by i
  • makeNotVar(i) creates the negated variable indexed by i
These basic BDDs can be combined by using the methods of the BDD interface to represent complex formulas.

Beedeedee is API-compatible with JavaBDD. Existing applications can be ported by using the adapter factory JavaBDDAdapterFactory.
